# Sawtak Blockchain Integration Guide

> Backend ↔ Cosmos SDK — using **raw CosmJS + REST** (no ts-client dependency).

---

## Architecture

```
┌──────────────┐    fetch (REST)     ┌──────────────┐
│  Elysia API  │ ──────────────────► │  Cosmos LCD  │  ← Queries (read)
│  (Bun)       │    port 1317        │  (REST API)  │
└──────┬───────┘                     └──────────────┘
       │                                    ▲
       │   CosmJS signAndBroadcast          │
       │        port 26657                  │
       └──────────────────────────► ┌───────┴──────┐
                                    │   sawtakd     │  ← Transactions (write)
                                    │   (node)      │
                                    └──────────────┘
```

| Port   | Protocol | Use            |
|--------|----------|----------------|
| `1317` | REST     | Read (queries) |
| `26657`| RPC      | Write (sign & broadcast txs) |

## Operation 1: Submit Anonymous Complaint (Write)

Uses CosmJS to sign and broadcast a transaction via RPC.

```typescript
import { DirectSecp256k1HdWallet, Registry } from "@cosmjs/proto-signing";
import { SigningStargateClient, StdFee } from "@cosmjs/stargate";

const RPC_URL = process.env.COSMOS_RPC_URL || "http://localhost:26657";

// ── Type URL (from proto) ──
const TYPE_URL_ANONYMOUS = "/sawtak.sawtak.v1.MsgSubmitAnonymousComplaint";

// ── 1. Create wallet from mnemonic ──
const wallet = await DirectSecp256k1HdWallet.fromMnemonic(
  process.env.RELAYER_MNEMONIC!,
  { prefix: "sawtak" }
);
const [account] = await wallet.getAccounts();

// ── 2. Connect signing client ──
const signingClient = await SigningStargateClient.connectWithSigner(
  RPC_URL,
  wallet
);

// ── 3. Build the message ──
const msg = {
  typeUrl: TYPE_URL_ANONYMOUS,
  value: {
    creator:      account.address,
    trackingId:   "SAWTAK-ABC12345",
    title:        "Water supply issue",
    text:         "sha256-hash-of-complaint-content",
    category:     "infrastructure",
    area:         "cairo",
    directedTo:   "ministry",
    incidentDate: "2025-03-15",
    evidence:     "ipfs://QmXyz...",
    proof:        "",              // ZK proof placeholder
  },
};

// ── 4. Sign & broadcast ──
const fee: StdFee = { amount: [], gas: "200000" };
const result = await signingClient.signAndBroadcast(
  account.address,
  [msg],
  fee
);

console.log("Tx Hash:", result.transactionHash);
console.log("Success:", result.code === 0);
```

### Message fields reference

| Field          | Required | Description              |
|----------------|----------|--------------------------|
| `creator`      | ✅       | Signer address (elysia_relayer) |
| `trackingId`   | ✅       | UUID/code generated by backend |
| `title`        | ✅       | Complaint title          |
| `text`         | ✅       | Hash of the real content |
| `category`     | ✅       | e.g. infrastructure, roads |
| `area`         | ✅       | e.g. cairo, giza         |
| `directedTo`   | ✅       | ministry, university, ngo |
| `incidentDate` | ✅       | ISO date string          |
| `evidence`     | ✅       | IPFS CID                 |
| `proof`        | ✅       | ZK proof (empty if unused) |

> **Note:** `MsgSubmitIdentifiedComplaint` is the same but without the `proof` field. Type URL: `/sawtak.sawtak.v1.MsgSubmitIdentifiedComplaint`

## Operation 2: Audit / Read Public Complaints (Read)

Uses plain `fetch` to the REST API. **Zero dependencies needed.**

### List all complaints

```typescript
const API_URL = process.env.COSMOS_API_URL || "http://localhost:1317";

async function listComplaints(limit = 20, offset = 0) {
  const res = await fetch(
    `${API_URL}/sawtak/v1/complaint?pagination.limit=${limit}&pagination.offset=${offset}`
  );
  const data = await res.json();
  return data;
  // data.complaint  → Complaint[]
  // data.pagination → { next_key, total }
}
```

### Get single complaint

```typescript
async function getComplaint(id: number) {
  const res = await fetch(`${API_URL}/sawtak/v1/complaint/${id}`);
  const data = await res.json();
  return data.complaint;
}
```

### Response shape

```json
{
  "complaint": {
    "id": "1",
    "tracking_id": "SAWTAK-ABC12345",
    "complaint_type": "anonymous",
    "title": "Water supply issue",
    "text": "sha256hash...",
    "category": "infrastructure",
    "area": "cairo",
    "directed_to": "ministry",
    "incident_date": "2025-03-15",
    "evidence": "ipfs://QmXyz...",
    "status": "submitted"
  }
}
```

### REST endpoints summary

| Endpoint                        | Method | Description        |
|---------------------------------|--------|--------------------|
| `/sawtak/v1/complaint`          | GET    | List (paginated)   |
| `/sawtak/v1/complaint/{id}`     | GET    | Get by ID          |
| `/sawtak/v1/params`             | GET    | Module params      |

## Operation 3: Update Complaint Status (Write)

✅ **Now on-chain** — `MsgUpdateComplaintStatus` is live.

Type URL: `/sawtak.sawtak.v1.MsgUpdateComplaintStatus`

```typescript
const TYPE_URL_UPDATE_STATUS = "/sawtak.sawtak.v1.MsgUpdateComplaintStatus";

const msg = {
  typeUrl: TYPE_URL_UPDATE_STATUS,
  value: {
    creator:     account.address,   // signer (elysia_relayer)
    id:          42,                // on-chain complaint ID (uint64)
    newStatus:   "under_review",    // or "in_progress", "resolved", "rejected"
    publicNotes: "Assigned to ministry team",
  },
};

const result = await signingClient.signAndBroadcast(
  account.address,
  [msg],
  fee
);
```

### Message fields

| Field         | Type   | Description                   |
|---------------|--------|-------------------------------|
| `creator`     | string | Signer address                |
| `id`          | uint64 | On-chain complaint ID         |
| `newStatus`   | string | New status value              |
| `publicNotes` | string | Optional admin note           |

### Status flow

```
submitted → under_review → in_progress → resolved
                                       → rejected
```

## CLI Quick Reference

```bash
# Start chain
ignite chain serve

# Submit anonymous complaint
sawtakd tx sawtak submit-anonymous-complaint \
  "SAWTAK-ABC123" "Title" "sha256hash" "category" \
  "cairo" "ministry" "2025-03-15" "ipfs://Qm..." "" \
  --from elysia_relayer --yes

sawtakd q sawtak list-complaint

  --from elysia_relayer --yes

curl http://localhost:1317/sawtak/v1/complaint
curl http://localhost:1317/sawtak/v1/complaint/1
```

## Error Handling

```typescript
try {
  const result = await submitAnonymousComplaint(data);
  if (result.code !== 0) {
    console.error("Chain rejected:", result.rawLog);
    return;
  }
  // Save result.transactionHash to Prisma DB
} catch (err) {
  console.error("Broadcast failed:", err);
}
```

| Error | Cause | Fix |
|-------|-------|-----|
| `account not found` | No tokens | Fund the account |
| `insufficient fees` | Low gas | Set gas to `"300000"` |
| `Signer is not present` | No mnemonic | Check `.env` |
| `key not found` | Invalid complaint ID | Verify ID exists |
| `connection refused` | Node down | Start chain first |
